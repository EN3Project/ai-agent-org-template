# Development Examples

These examples show how to invoke the Development preset without adding process weight.

Use the brief as a task shape, not as a required form. A normal user request can be shorter when the context is obvious.

Suggested route:

- Small local fix: Developer -> Reviewer.
- Normal feature or bugfix: Architect -> Developer -> Tester -> Reviewer.
- Broad or risky change: Architect -> user checkpoint -> Developer -> Tester -> Reviewer.
