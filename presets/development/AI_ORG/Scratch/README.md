# Scratch

Temporary workspace for active development tasks.

Use only when a task benefits from a short handoff note between Architect, Developer, Tester, and Reviewer.

Rules:

- Do not store secrets, credentials, personal data, customer data, or proprietary raw content.
- Do not treat Scratch as long-term memory.
- Delete or ignore task notes once they stop being useful.
- Prefer concise summaries over copied source or logs.
